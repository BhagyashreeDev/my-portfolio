Problem

Managing schedules through static lists leads to data synchronization issues. There was a need for a reactive interface that handles real-time updates, inline editing, and state persistence without page reloads.

Approach

  • Implemented State Lifting to maintain a 'Single Source of Truth' in the parent App component
  • Developed controlled form components utilizing React Hooks for synchronized input handling
  • Created a dynamic CRUD logic for adding, deleting, and bulk-clearing appointments
  • Designed an inline editing toggle using local UI state to switch between text and input modes

Tools & Technologies

  • React.js (Functional Components)
  • JavaScript ES6 (Spread Operators & Closures)
  • React Hooks (useState)
  • CSS3 (Responsive Table Layouts)

Key Insights

  • Lifting state ensures data consistency across the Form and List sibling components
  • Immutable state updates (using spread operators) prevent side effects and render bugs
  • Local UI state for 'editedIndex' optimizes performance by limiting edit-mode to one row at a time
  • Form event prevention (preventDefault) ensures a seamless Single Page Application experience

Challenges & Learnings

  • Mastered the use of map() and splice() patterns for immutable array manipulation
  • Learned to manage memory efficiency by clearing input states after successful submissions
  • Improved skills in conditional rendering for complex UI transitions like Save/Cancel modes
