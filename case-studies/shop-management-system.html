Problem

Manual inventory tracking and unrestricted user access led to data inconsistency and security risks. Existing workflows lacked a structured way to manage relational data and role-based permissions.

Approach

  • Developed a modular layered architecture separating View, Model, and Utility layers
  • Implemented Hibernate 6 ORM to map Java objects to MySQL relational tables
  • Designed a secure login system with specific ADMIN and USER access levels
  • Utilized HQL (Hibernate Query Language) for database-independent real-time filtering

Tools & Technologies

  • Java 17 & Java Swing (AWT)
  • Hibernate 6 (ORM)
  • MySQL 8 Database
  • Maven (Build Tool)

Key Insights

  • Bidirectional @OneToOne mapping with CascadeType.ALL ensures data integrity
  • Context-aware UI state management prevents conflicting CRUD operations
  • HQL LIKE queries enable instant search results without database dependency
  • Secure credential handling via .properties files prevents sensitive data leaks

Challenges & Learnings

  • Mastered memory management by using .dispose() during frame transitions
  • Learned to implement role-based access control (RBAC) at the application level
  • Improved understanding of relational database schemas and foreign key constraints
